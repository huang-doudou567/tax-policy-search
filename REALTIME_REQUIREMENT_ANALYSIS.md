## 逐项重评估

### #2 金税四期指标速查表 —— 🔴 需要彻底改设计

**原方案**：`references/tax_risk_indicators.md` 静态 MD 文件，列出 200+ 指标

**问题**：
- 金税四期指标体系在 2023-2024 年快速迭代，静态表 3 个月就过时
- 各地税务机关可能有地方性补充指标
- 用户看到"静态表"会误以为这就是当前全部指标

**新方案**：
```
用户问"金税四期有什么风险指标"
    ↓
第一步：NPC 全文搜索 "金税四期" / "税收风险" / "税务稽查"（实时）
    ↓
第二步：chinatax.gov.cn 搜索最新公告（实时）
    ↓
第三步：AnySearch 补充行业动态（实时）
    ↓
输出格式：
    [实时搜索结果]（最新相关法规）
    +
    [指标框架参考]（标注"框架仅供参考，以实时搜索结果为准"）
```

**静态部分降级为"搜索提示词"**，不再作为独立知识源：
```markdown
# references/tax_risk_framework.md  ← 改名
## 仅作为搜索关键词提示，非权威知识
- 增值税: 虚开发票、骗取留抵退税、进项税抵扣异常...
- 企业所得税: 两税收入差异、长亏不倒、税负率异常...
- ...
> ⚠️ 以上仅为搜索提示，实际指标以实时查询 NPC + chinatax + AnySearch 结果为准
```

**成本变化**：原 1h → 新 2h（搜索策略设计 + 框架整理 + 强制验证标注）

### #6 缓存策略 —— 🟡 需要调整 TTL

**原方案**：照搬 npc-law-db 的 1h/24h/7d

**问题**：税收政策随时可能发布新公告，1 小时缓存意味着用户可能看到过期数据

**新方案**：

| 数据类型 | 原 TTL | 新 TTL | 理由 |
|---------|--------|--------|------|
| 搜索结果 | 1h | **5min** | 政策公告随时更新 |
| 详情元数据 | 24h | **1h** | 法规元数据变动少，但时效性状态可能变化 |
| DOCX 文件 | 7d | **24h** | 法规全文几乎不变，但可能出新版 |
| 金税指标查询 | N/A | **no-cache** | 必须实时 |

```python
# 默认禁用缓存，用 --cache 显式开启
python scripts/tax_search.py "增值税" --size 20           # 无缓存
python scripts/tax_search.py "增值税" --size 20 --cache   # 开启 5min 缓存
```

**额外要求**：搜索结果头部必须标注查询时间戳：
```
> 🔍 实时查询 | 2026-06-29 15:23:45 | 数据源: NPC 国家法律法规数据库
```

### #1 chinatax WebFetch —— 🟢 不变但优先级提升

原方案把它列为"兜底"，但在"实时更新"要求下，chinatax.gov.cn 的重要性显著提升——它是**部门规章和最新公告的唯一实时来源**。

优先级调整为：
```
1. NPC API —— 法律/行政法规层（权威性最高，实时）
2. chinatax.gov.cn —— 部门规章/公告/解读（时效性最强，实时）  ← 从"兜底"升级为"必需"
3. AnySearch —— 行业动态/地方政策补充（覆盖面最广，实时）
```

## 修正后的 SKILL.md 核心设计原则

原方案强调"知识库"模式（参考 tax-policy-knowledge），但在"实时更新"下必须切换为"搜索引擎"模式（参考 npc-law-db）：

```
❌ 旧设计（静态知识库）：
   SKILL.md 内嵌政策问答示例 → AI 凭记忆/训练数据回答

✅ 新设计（实时搜索引擎）：
   SKILL.md 描述"如何搜索"而非"答案是什么"
   每次用户提问 → 强制触发至少一次实时 API 调用
   训练数据仅用于理解用户意图，不用于提供答案
```

具体体现在 SKILL.md 的修改：

| 模块 | 旧设计 | 新设计 |
|------|--------|--------|
| 意图分类表 | 保留 | 保留（判断搜索策略用） |
| 问答示例 | 内嵌静态答案 | **删除**，改为"搜索命令示例" |
| 回答模板 | 保留四段式 | 增加"数据获取时间"字段 |
| 知识原则 | "优先引用最新政策" | **"禁止使用训练数据直接回答，必须先查 API"** |
| 禁止行为 | 保留 4 条 | **新增：禁止不查询就回答** |
