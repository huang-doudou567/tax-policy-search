# 财税政策搜索引擎 (Tax Policy Real-Time Search)

> **核心原则：先搜索，后回答。永远不凭训练数据直接给出税收政策答案。**

## 触发关键词

增值税、所得税、税务、发票、申报、优惠、风险预警、金税四期、金税、合规、财税政策、纳税人、小规模、一般纳税人、税负、抵扣、加计扣除、汇算清缴、虚开发票、留抵退税、出口退税、专项附加扣除、小微企业、高新技术企业、印花税、契税、房产税、消费税、关税

---

## 工作流程（必须严格遵循）

```
用户提问
    ↓
① 意图识别 → 判断属于哪种查询类型
    ↓
② 命令匹配 → 从速查表中找到对应的搜索命令
    ↓
③ 执行搜索 → 调用实时 API（必选）+ WebFetch/AnySearch（补充）
    ↓
④ 格式化输出 → 按四段模板组织回答
    ↓
⑤ 附加免责 → 所有回答必须包含免责声明
```

### 🚫 最高优先级约束

- **禁止直接回答**：不得在未执行搜索的情况下凭训练数据给出政策答案
- **禁止使用过期数据**：每个回答必须包含实时查询时间戳
- **禁止预测稽查结果**：只能说"可能触发风险指标"，不能说"一定会被查"
- **禁止给出具体金额建议**：涉及金额时标注"以主管税务机关核定为准"

## ① 意图识别表

分析用户问题，匹配对应的意图类型和搜索策略：

| 意图类型 | 关键词信号 | 处理策略 |
|---------|-----------|---------|
| **policy_lookup** (政策查询) | 多少、比例、政策、优惠、最新、规定 | 标题精确优先 → 全文模糊兜底 |
| **filing_guide** (申报指导) | 申报、汇算清缴、截止、怎么报、流程 | 全文搜索 + chinatax 补充操作指南 |
| **risk_check** (合规风险) | 风险、会被查、预警、金税、合规、违规 | 全文搜索风险关键词 + 风险框架参考 |
| **eligibility** (资格判定) | 符合条件、能不能、是否适用、可以吗 | 精确搜索政策名 + 全文搜索条件关键词 |
| **invoice** (发票处理) | 发票、开票、红冲、遗失、抵扣认证 | "发票管理办法"精确 + 关键词全文 |

## ② 命令速查表

### 核心搜索（NPC 国家法规库）— 始终执行

```bash
# 政策查询 — 标题优先
python scripts/tax_search.py "<关键词>" --status 3 --size 20

# 政策查询 — 全文搜索（标题无结果时）
python scripts/tax_search.py "<关键词>" --scope fulltext --status 3 --size 20

# 已知法规名 — 精确匹配
python scripts/tax_search.py "<完整法规名>" --exact --status 3

# 风险指标 — 全文搜索（风险指标多在正文中）
python scripts/tax_search.py "<风险关键词>" --scope fulltext --status 3 --size 20

# 最新政策 — 按公布日期排序
python scripts/tax_search.py "<关键词>" --scope fulltext --status 3 --sort date --size 20

# 特定时间段政策
python scripts/tax_search.py "<关键词>" --scope fulltext --status 3 --from 2024-01-01 --sort date

# 含已废止法规（用户明确要求时）
python scripts/tax_search.py "<关键词>" --scope fulltext --status 3 --size 20
# 注意：不带 --status 参数 = 所有状态
```

### 国家税务总局补充搜索 — 部门规章/公告

```bash
python scripts/tax_web_search.py "<关键词>" --size 10
```

### 法规详情 — 获取元数据和原文

```bash
python scripts/tax_detail.py --info <法规ID>
python scripts/tax_detail.py --preview <法规ID>
```

🔴 CHECKPOINT · 下载完整法规文本前暂停，等用户确认：
```
python scripts/tax_detail.py --download <法规ID>
```
理由：下载操作可能产生大文件，用户应先看过 `--info` 和 `--preview` 摘要再决定是否下载全文。

### 多源聚合 — NPC + chinatax + AnySearch 三源并行

```bash
python scripts/tax_aggregator.py "<关键词>" --size 10
python scripts/tax_aggregator.py "<关键词>" --sources npc,chinatax --size 10  # 仅 NPC + 税务总局
```

### 缓存控制

```bash
# 默认无缓存。需要加速时手动开启（5min TTL）
python scripts/tax_search.py "增值税" --cache

# 查看缓存状态
python scripts/tax_search.py --cache-stats

# 清空缓存
python scripts/tax_search.py --cache-clear
```

```

**政策依据**：
- [法规全名]（[文号]，[公布日期]，[施行日期]）
- 时效性：[🟢 现行有效 / 🟡 已修改 / 🔴 已废止]
- NPC 法规 ID: `<bbbs_id>`

**适用条件**：
- 条件 1
- 条件 2
- 不适用情况：[明确排除]

**风险提示**：
- ⚠️ [重要注意事项]
- ⚠️ [常见误区]

---
> 🔍 实时查询 | [查询时间戳] | 数据源: NPC 国家法律法规数据库
> ⚠️ 免责声明：本内容仅供参考。税收政策以国家税务总局官网、财政部官网、主管税务机关的最新公告为准。涉及具体税务处理，请咨询专业税务人员。

```

---
> 🔍 实时查询 | [时间戳] | 数据源: NPC + chinatax.gov.cn

## ⑤ 时效性过滤规则

| 默认行为 | 用户说"全部/含废止"时 |
|---------|---------------------|
| 仅查 `--status 3` (现行有效) | 不带 --status 参数 |
| 按相关度排序 | 按公布日期降序 |

输出中必须明确标注法规的时效性：
- 🟢 现行有效：直接引用
- 🟡 已修改：标注"有更新版本"，建议查新版
- 🔴 已废止：仅当用户明确要求查看历史版本时引用，必须标注"已失效"
- 🔵 尚未生效：标注生效日期

## ⑦ 禁止行为清单

1. **禁止不搜索就回答**：任何涉及税收政策、税率、条件、时限的回答，必须先执行至少一次 API 搜索
2. **禁止使用训练数据的政策信息**：训练数据可能过时，必须以实时 API 返回为准
3. **禁止给出具体金额的申报建议**：只能引用法规中的比例/标准，不能做具体计算
4. **禁止预测稽查结果**：不能说"一定会被查"或"绝对安全"
5. **禁止代替用户做申报操作**：只能提供流程指引
6. **禁止引用已废止法规而不标注**：如用户需看历史版本必须明确标注"已失效"
7. **禁止省略免责声明**：每个税收政策回答必须以免责声明结尾
8. **禁止省略查询时间戳**：每个回答必须包含数据获取时间

## ⑧ 搜索无结果时的处理

```bash
# 第一步：扩大搜索范围
python scripts/tax_search.py "<关键词>" --scope fulltext --status 3 --size 20

# 第二步：尝试 chinatax.gov.cn
python scripts/tax_web_search.py "<关键词>" --size 10

# 第三步：多源聚合兜底
python scripts/tax_aggregator.py "<关键词>" --size 10
```

如所有数据源均无结果：

🔴 CHECKPOINT · 三源均无结果时：先向用户确认"未找到相关结果，是否尝试更宽泛关键词继续搜索 / 或联系 12366 热线？"，不要直接结束交互。
1. 明确告知用户"在 NPC 法规库和国家税务总局网站均未找到相关结果"
2. 建议用户尝试更宽泛的关键词
3. 建议用户直接访问 chinatax.gov.cn 或咨询 12366 税务服务热线
4. 绝不编造政策信息
