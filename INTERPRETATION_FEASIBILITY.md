# 政策解读窗口 — 操作复杂性评估

## 目标

在法规弹窗中增加 AI 政策解读功能：用户查看法规原文时，一键生成通俗化解读。

---

## 总体评估

```
复杂度: ⭐⭐⭐ 中等
后端新增: ~120 行
前端新增: ~150 行
LLM 接入: 取决于方案选择
总工时: 4-8h（取决于选哪个方案）
```

---

## 三个方案的详细对比

### 方案 A：复用 Claude Code（零依赖，2h）

```
用户点击"AI 解读" → 后端调用 Claude Code CLI → Claude 读取法规全文 → 生成解读 → 返回前端
```

| 维度 | 评估 |
|------|------|
| **实现方式** | Flask 调用 `claude --print "请解读以下法规..." ` 子进程 |
| **新增依赖** | 无（Claude Code 已安装） |
| **后端代码** | ~40 行 — 一个 `/api/interpret/<bbbs_id>` 端点 |
| **前端代码** | ~80 行 — 法规弹窗中加一个"AI 解读"按钮 + 解读结果面板 |
| **延迟** | 3-10 秒（取决于模型和法规长度） |
| **成本** | 消耗 Claude API token |
| **质量** | ⭐⭐⭐⭐⭐ — Claude 的法律理解能力最强 |
| **适用场景** | **demo / 个人使用** |

**前端交互流**：
```

│ 📜 增值税法                           │
│ ┌──────────────┬───────────────────┐  │

│ │              │                   │  │

```

---

### 方案 B：接入外部 LLM API（需 API Key，3h）

```
用户点击"AI 解读" → 后端调 Anthropic/OpenAI API → 返回解读
```

| 维度 | 评估 |
|------|------|
| **实现方式** | 使用 Anthropic SDK 或 OpenAI SDK，直接调 API |
| **新增依赖** | `pip install anthropic` 或 `pip install openai` |
| **后端代码** | ~80 行 — `/api/interpret` 端点 + Prompt 模板 |
| **前端代码** | ~80 行（同方案 A） |
| **配置要求** | 用户需设置 `ANTHROPIC_API_KEY` 或 `OPENAI_API_KEY` 环境变量 |
| **延迟** | 2-5 秒 |
| **成本** | Anthropic: ~$0.01-0.05/次; OpenAI: ~$0.005-0.02/次 |
| **质量** | ⭐⭐⭐⭐ — 取决于模型选择 |
| **适用场景** | **生产环境 / 多人使用** |

---

### 方案 C：本地模型（需安装，5h）

```
用户点击"AI 解读" → 后端调 Ollama/LM Studio 本地 API
```

| 维度 | 评估 |
|------|------|
| **实现方式** | 调 Ollama API (`http://localhost:11434/api/generate`) |
| **新增依赖** | 需安装 Ollama + 下载模型（qwen2.5 或 llama3 等中文模型） |
| **后端代码** | ~80 行 |
| **前端代码** | ~80 行（同方案 A） |
| **配置要求** | 需安装 4-8GB 的中文 LLM 模型 |
| **延迟** | 5-30 秒（取决于硬件和模型大小） |
| **成本** | 免费（仅电费和硬件） |
| **质量** | ⭐⭐⭐ — 取决于模型，中文法律理解弱于 Claude |
| **适用场景** | **完全离线 / 隐私敏感** |

## 后端新增代码量（方案 A）

```python
# tax_server.py 新增

@app.route("/api/interpret/<bbbs_id>", methods=["POST"])
def api_interpret(bbbs_id):
    """Generate AI interpretation for a law."""
    data = request.get_json() or {}
    keyword = data.get("keyword", "")
    sections_text = data.get("sections_text", "")  # 前端传法规全文
    
    if not sections_text:
        return jsonify({"error": "sections_text required"}), 400
    
    # 构建 Prompt
    prompt = f"""你是一位资深税务专家。请用通俗易懂的中文解读以下法规条文。
    
用户关心的关键词：{keyword or "无特定关键词"}

法规全文：
{sections_text[:8000]}  # 截断到 8000 字

请按以下格式输出 Markdown：

## 适用主体
[谁适用这个法规]

## 核心要点
- 要点1
- 要点2

## {keyword + "相关条款" if keyword else "关键条款"}
[与用户关心的关键词最相关的条款]

## 注意事项
- ⚠️ 注意点1
- ⚠️ 注意点2

要求：语言通俗，面向企业主/财务人员，不要直接重复法条原文。"""
    
    # 调用 Claude Code CLI
    result = subprocess.run(
        ["claude", "--print", prompt],
        capture_output=True, text=True, timeout=60,
        env={**os.environ, "CLAUDE_CODE_SKIP_SKILLS": "1"}
    )
    
    if result.returncode != 0:
        return jsonify({"error": result.stderr}), 500
    
    return jsonify({"interpretation": result.stdout, "model": "claude"})
```

**约 40 行**，核心就是构建 Prompt + 调用 `claude --print`。

---

## 风险项

| 风险 | 等级 | 缓解 |
|------|------|------|
| Claude Code 子进程超时 | 低 | 设 60s timeout + 前端 loading 状态 |
| 长法规超出 token 限制 | 中 | 前端传摘要/截断; 或只传选中章节 |
| 解读质量不稳定 | 低 | Prompt 中有明确的输出格式约束 |
| 用户没有 API Key 无法用方案 B | — | 先用方案 A，后续再加 |

---

## 一句话总结

**方案 A 只需 2 小时、约 120 行代码，在当前环境下立即可落地。** 核心就是 Flask 调用 `claude --print` 子进程，前端加一个右侧解读面板。方案 B/C 可以作为后续的可选升级路径。
