## 1. 项目概述

### 1.1 项目定位

为企业和个人提供**权威、实时、智能**的中国税收政策法规查询与解读服务，解决当前市场上税务信息查询的三大痛点：

| 痛点 | 现状 | 本项目方案 |
|------|------|-----------|
| **信息分散** | 法律在 NPC，公告在 chinatax，解读在财政部，用户需逐个网站查找 | 三源实时聚合，一次搜索覆盖所有层级 |
| **时效难辨** | 搜索结果混杂已废止法规，用户无法判断哪些仍然有效 | 默认仅查现行有效，自动标注废止/修改状态 |
| **解读缺失** | 法条原文晦涩，官方解读分散，缺少通俗化解释 | 官方解读搜索 + AI 通俗解读双保险 |

### 1.2 目标用户

```
┌──────────────┬──────────────────────────────────┐
│ 用户角色       │ 典型场景                          │
├──────────────┼──────────────────────────────────┤
│ 企业主         │ "小微企业增值税有什么优惠？"         │
│ 财务人员       │ "研发费用加计扣除最新比例是多少？"    │
│ 个体工商户     │ "小规模纳税人开发票的限额？"         │
│ 代理记账/税务师 │ "最新企业所得税优惠政策汇总"        │
│ 法务/合规      │ "金税四期关联交易风险指标"          │
└──────────────┴──────────────────────────────────┘
```

### 1.3 核心原则

> **先搜索，后回答。永远不凭训练数据直接给出税收政策答案。**

每一项政策回答必须：
- 附带实时查询时间戳
- 标注数据来源（NPC 法规库 / 国家税务总局 / 财政部等）
- 包含法规唯一标识（NPC bbbs ID）
- 标注时效性状态（🟢现行有效 / 🟡已修改 / 🔴已废止）

### 1.4 项目规模

### 2.2 关键架构决策

| # | 决策 | 选择 | 为什么 |
|---|------|------|--------|
| 1 | 主数据源 | NPC `flk.npc.gov.cn` API | 全国人大官网，权威性最高，有完整 REST API |
| 2 | 部门规章搜索 | Bing `site:` 搜索 + WebSearch | chinatax 无公开 API，且各省搜索系统不统一 |
| 3 | DOCX 解析 | Python stdlib (zipfile + ElementTree) | 零 pip 依赖，避免 python-docx 的安装复杂性 |
| 4 | 深度学习框架 | 不使用 | 场景不需要 ML，意图分类用规则引擎已足够准确 |
| 5 | 前端框架 | 纯 HTML/CSS/JS，零框架 | 单文件部署，无构建步骤，所有功能在一个 412 行文件中完成 |
| 6 | 缓存策略 | 默认关闭，可选开启（5min TTL） | 税收政策随时更新，缓存可能返回过时数据 |
| 7 | 前端实时 | 每次交互都调 API | 不做本地缓存层，所有数据来自实时查询 |
| 8 | AI 解读 | Claude Code CLI 子进程 | 当前环境已安装，零新依赖 |
| 9 | 技能格式 | Anthropic SKILL.md | Claude Code 自动发现和加载 |
| 10 | 省市税局 | Bing site: 间接路由 | 36 省无统一 API，搜索引擎是最稳定的抽象层 |

## 3. 数据流与交互流程

### 3.1 用户搜索全链路

```
时间轴 →

0ms     用户输入 "小微企业增值税优惠"
         │
50ms    意图识别引擎
         │ detect_intent("小微企业增值税优惠") → "policy_lookup"
         │ resolve_tax_type("小微企业增值税优惠") →  {"type": "企业所得税", "aliases": [...]}
         │
100ms   策略匹配
         │ 意图: policy_lookup → 策略: 标题模糊优先
         │ 税种: 企业所得税 → 搜索关键词: "企业所得税" + "小微企业" + "优惠"
         │
200ms   API 调用
         │ POST https://flk.npc.gov.cn/law-search/search/list
         │ payload: {searchRange: 1, searchType: 2, searchContent: "小微企业",
         │           sxx: [3], pageSize: 20}
         │
800ms   API 返回
         │ {total: 538, rows: [{bbbs: "xxx", title: "企业所得税法", sxx: 3, ...}]}
         │
900ms   结果格式化
         │ tax_formatter.py → 四段式 Markdown
         │   · 时效性徽章 (🟢/🟡/🔴)
         │   · 查询时间戳
         │   · 免责声明
         │
1000ms  返回前端 → 渲染结果卡片
         │
         ▼
用户点击某个结果卡片 → 法规弹窗
         │
         ├── Tab 1: 📖 法规原文
         │      GET /api/text/{bbbs_id}
         │      → 下载 DOCX → stdlib 解析 → 118段/38条法条
         │      → 关键词 "增值" "小微" 黄色高亮
         │
         ├── Tab 2: 🔍 官方解读
         │      GET /api/interpretations/{bbbs_id}?keyword=...
         │      → ThreadPoolExecutor 并行搜 4 个站点
         │      → Bing site:chinatax.gov.cn + site:mof.gov.cn + site:gov.cn
         │      → 返回 3-12 条官方解读链接
         │
         ├── Tab 3: 🤖 AI 解读
         │      GET /api/ai-interpret/{bbbs_id}?keyword=...
         │      → 提取法规前 30 条法条 → 构建 Prompt
         │      → Claude Code CLI 子进程 生成通俗解读
         │      → 返回：适用主体 / 核心要点 / 注意事项
         │
         └── Tab 4: 🌐 相关网页
                → AnySearch legal domain 补充搜索
```

## 8. 安全与合规

### 8.1 8 条禁止行为

| # | 规则 | 原因 |
|---|------|------|
| 1 | 禁止不搜索就回答 | 税收政策可能随时更新 |
| 2 | 禁止使用训练数据直接提供政策信息 | 训练数据有时效落差 |
| 3 | 禁止给出具体金额的申报建议 | 只有税务机关有权核定 |
| 4 | 禁止预测稽查结果 | 稽查决策权在税务机关 |
| 5 | 禁止代替用户做申报操作 | 法律责任由纳税人承担 |
| 6 | 禁止引用已废止法规而不标注 | 可能误导用户 |
| 7 | 禁止省略免责声明 | 法律合规要求 |
| 8 | 禁止省略查询时间戳 | 确保时效可追溯 |

### 8.2 强制输出要素

每个税收政策回答必须包含：
```
🔍 数据来源标注：NPC 国家法规数据库 / 国家税务总局 / etc.
🕐 查询时间戳：YYYY-MM-DD HH:MM:SS
🟢/🟡/🔴 时效性徽章
📄 法规 NPC ID
⚠️ 免责声明：仅供参考，以官方最新公告为准
```

### 8.3 免责声明模板

```
> ⚠️ 免责声明：本内容仅供参考。税收政策以国家税务总局官网 (chinatax.gov.cn)、
> 财政部官网 (mof.gov.cn)、主管税务机关的最新公告为准。
> 涉及具体税务处理，请咨询专业税务人员。
```

## 12. 部署方案

### 12.1 本地开发部署

```bash
# 1. 安装依赖
pip install flask requests urllib3

# 2. 启动
cd tax-policy-search
python scripts/tax_server.py

# 3. 访问
http://localhost:5080
```

**特点**：
- 单进程，Flask 开发服务器
- 不依赖外部数据库
- 所有数据来自实时 API 查询
- 适合个人使用和 Demo 演示

### 12.2 生产部署建议

| 组件 | 建议方案 |
|------|---------|
| WSGI | Gunicorn / uWSGI |
| 反向代理 | Nginx（静态文件 + API 代理） |
| 进程管理 | systemd / Supervisor |
| 日志 | Python logging → 文件轮转 |
| 监控 | /health 端点 + Prometheus |
| 并发 | 多 worker 进程（每个 worker 独立缓存） |
| 安全 | HTTPS + API Key 认证（如需对外开放） |

### 12.3 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `TAX_SEARCH_VERIFY_SSL` | NPC API SSL 验证 | `0`（关闭） |
| `NPC_LAW_NO_CACHE` | 禁用 NPC API 缓存 | `0`（启用） |
| `ANYSEARCH_API_KEY` | AnySearch API Key | 空（匿名模式） |

| 指标 | 数值 |

## 附录 B：已知限制

| 限制 | 说明 | 缓解 |
|------|------|------|
| NPC API 不返回全文 | 需二次调用下载 API | 已在 tax_detail.py 中自动化 |
| chinatax.gov.cn 无公开 API | 依赖 Bing 搜索引擎 | 搜索结果可能有延迟 |
| 部分省市税局网站不对外开放 | 无法通过搜索引擎获取 | 明确告知用户直接访问该站 |
| AI 解读依赖 Claude Code CLI | 需本地安装 | 官方解读 Tab 作为首选，AI 为兜底 |
| DOCX 下载有时效性签名 | URL 约 1 小时后过期 | 从不下缓存 URL，每次实时获取 |
